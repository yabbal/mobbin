## AuthResource

The `auth` resource handles the connection to Mobbin via Supabase Auth.

### checkEmail

Checks the authentication method available for an email. Some accounts use a password, others use a magic link.

```typescript
const type = await client.auth.checkEmail("user@example.com");
// "password" | "magic_link"

if (type === "magic_link") {
  console.log("This account uses a magic link (not supported by the SDK)");
}
```

### login

Logs in the user and returns a complete Supabase session.

```typescript
const session = await client.auth.login("user@example.com", "password");

console.log(session.access_token); // JWT token
console.log(session.expires_at);   // Expiration timestamp (seconds)
console.log(session.user.id);      // User UUID
console.log(session.user.email);   // Email
```

The session contains an `access_token` (JWT, expires after 1h) and a `refresh_token` (long-lived).

### refresh

Refreshes the session with the refresh token.

```typescript
const newSession = await client.auth.refresh(session.refresh_token);
// New session with a new access_token
```

### getUser

Retrieves the information of the logged-in user.

```typescript
const user = await client.auth.getUser();
console.log(user.id, user.email);
```
